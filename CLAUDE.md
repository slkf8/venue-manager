# 專案規則

## 技術規則
- 第一版使用純 HTML / CSS / JavaScript
- 不要改成 React
- 保持模塊化
- 不要重寫整體 UI
- 優先修復缺失核心與資料流問題
- 優先保持現有檔案命名與結構

## 產品規則
- 首頁核心是場地維護
- 頂部只有日期 + 打卡
- 儀表板最重要
- 儀表板分成設備區與消耗品區
- 設備在前，消耗品在後
- 設備進問題追蹤，消耗品不進問題追蹤
- 人數是次要資訊，放在巡查模塊

## 設備規則
- 狀態：正常 / 待處理 / 處理中 / 已處理
- 巡查中標記需要處理時，自動進入問題追蹤
- 已處理後需手動確認恢復正常
- 問題需記錄建立時間、已處理時間、恢復正常時間

## 消耗品規則
- 數量式：高於警戒值=正常，低於或等於警戒值=急需補充
- 狀態式：正常 / 偏少 / 急需補充
- 消耗品只做儀表板顏色提示，不進問題追蹤

## 歷史規則
- 歷史用底部 sheet 顯示
- 可直接編輯歷史
- 修改設備 / 消耗品歷史，要一路覆蓋到最新日期
- 修改人數 / 打卡歷史，只改當天

## 操作規則
- 所有刪除操作都要二次確認
- 不要做未被要求的重構
- 每次只改必要檔案
- 修改完成後要列出改了哪些檔案與原因
